# RealismThrift 项目说明文档

本项目是 RealismThrift 的数字化官网与内容管理系统，用于展示产品、接收询盘及维护博客。

---

## 一、项目核心组成

1.  **对外官网 (Next.js)**：提供给全球客户访问的前台界面，包含产品分类、公司介绍、合作流程及询盘表单。
2.  **内容后台 (Sanity Studio)**：供团队管理博客内容、SEO 信息，并支持 **Visual Editing（可视化编辑）** 进行实时预览。

---

## 二、当前主要能力

*   **全功能展示**：包含首页、关于我们、联系我们、订购流程、FAQ 及三大产品分类页。
*   **询盘系统**：集成表单安全校验与邮件自动发送功能，确保潜在客户咨询直达销售邮箱。
*   **博客管理**：支持图文排版、SEO 优化及分类管理。
*   **可视化预览**：在后台即可实时查看博客发布后的实际效果，降低编辑出错率。

---

## 三、重构与优化亮点

本项目基于原官网（realismthrift.com）进行深度重构，在保持品牌视觉一致性的基础上，实现了以下技术与业务优化：

1.  **视觉与细节升维**：保持与原站高度一致的专业视觉风格，并对全站 Icon（切换至 Lucide 库）、排版间距及响应式交互进行了精细化调整。
2.  **业务模块扩张**：在原有基础上新增了 **Used Clothes（旧衣）** 与 **Used Bag（旧包）** 核心产品模块，完善了批发业务版图。
3.  **内容生态构建**：独立开发了 **Blog（博客）** 模块，配套成熟的 **Headless CMS（Sanity Studio）** 后台，支持“即改即看”的 **Visual Editing（可视化编辑）** 功能。
4.  **数据与逻辑修正**：纠正了原站中多处排版混乱及数据冲突（如公司成立年限、出口国家统计不一致等逻辑错误），提升了品牌公信力。
5.  **交互功能实装**：将原站仅具装饰性的搜索框重构为 **全站全文检索系统**，支持产品、FAQ 及博客内容的实时搜索匹配。
6.  **SEO 深度优化**：
    *   全站页面均妥善配置了动态 `Metadata` 与 `OpenGraph` 属性。
    *   所有图片均根据业务场景编写了描述性的 `Alt Text`。
    *   自动生成 `sitemap.ts`、`robots.txt` 及专为 AI 引擎准备的 `llms.txt`。
    *   预留了 Facebook Pixel 与 Google Analytics 的接入接口。
7.  **高可靠询盘链路**：集成 **Resend** 邮件服务商，表单提交已通过实际生产测试。开发者只需在 `.env.local` 中配置 API KEY 与接收邮箱，即可实现 leads 秒级触达。

## 四、快速使用指南

### 1. 发布/修改博客
*   登录 Sanity 后台 -> 选择 `Post` -> 填写内容/上传图片 -> 点击右下角 **Publish**。
*   使用 **Presentation** 模式可进行“边看边改”。

### 2. 处理询盘
*   客户提交表单后，系统会自动发送邮件至 `sales@realismthrift.com`。若未收到，请检查.env.local文件配置情况。

## 五、项目结构与配置

### 1. 目录说明
*   `nextjs/`：官网前台代码。
*   `sanity/studio-realismthrift/`：内容管理后台代码。
*   `AGENTS.MD`：业务背景与团队架构。
*   `archive/`：历史审计与技术文档。

> [!IMPORTANT]
> 本项目强制使用 **pnpm** 进行包管理。请勿使用 `npm` 或 `yarn`，否则 `preinstall` 钩子将报错。

### 2. 环境配置项 (关键变量)
*   `RESEND_API_KEY`：邮件发送密钥。
*   `CONTACT_EMAIL`：接收询盘的邮箱。
*   `NEXT_PUBLIC_SANITY_PROJECT_ID`：Sanity 项目 ID。

## 六、注意事项与常见问题

*   **可视化编辑范围**：目前主要覆盖博客页面。首页等营销页面文案调整仍需技术协助。
*   **发布生效**：修改内容后必须点击 **Publish** 才能在前台看到更新。
*   **邮件未收到**：请优先检查垃圾邮件箱，或确认发件域名验证状态。
*   **预览失败**：确保本地开发环境时前后台同时运行。
